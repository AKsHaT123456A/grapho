# Deployment Guide

## Option 1: Vercel (Recommended - Free Tier Available)

### Steps:

1. **Push to GitHub**
   ```bash
   git remote add origin <your-github-repo-url>
   git push -u origin master
   ```

2. **Deploy to Vercel**
   - Go to [vercel.com](https://vercel.com)
   - Click "Import Project"
   - Select your GitHub repository
   - Configure:
     - Framework Preset: Next.js
     - Build Command: `npm run build`
     - Output Directory: `.next`
   
3. **Add Environment Variable**
   - In Vercel dashboard, go to Settings → Environment Variables
   - Add: `GEMINI_API_KEY` = `your_actual_api_key`
   - Redeploy

4. **Done!**
   - Your app will be live at `https://your-project.vercel.app`

### Notes:
- SQLite works on Vercel but data is ephemeral (resets on each deployment)
- For persistent data, consider upgrading to PostgreSQL (see below)

## Option 2: Railway (PostgreSQL Support)

### Steps:

1. **Create Railway Account**
   - Go to [railway.app](https://railway.app)
   - Sign up with GitHub

2. **Create New Project**
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Choose your repository

3. **Add PostgreSQL Database**
   - Click "New" → "Database" → "PostgreSQL"
   - Railway will auto-generate `DATABASE_URL`

4. **Update Prisma Schema**
   - Change `provider = "sqlite"` to `provider = "postgresql"`
   - Update `DATABASE_URL` in `.env`

5. **Add Environment Variables**
   - `GEMINI_API_KEY` = your key
   - `DATABASE_URL` = (auto-generated by Railway)

6. **Deploy**
   - Railway auto-deploys on push
   - Run migrations: `npx prisma migrate deploy`

## Option 3: Docker (Self-Hosted)

### Prerequisites:
- Docker installed
- Docker Compose (optional)

### Steps:

1. **Build Docker Image**
   ```bash
   docker build -t knowledge-graph-builder .
   ```

2. **Run Container**
   ```bash
   docker run -d \
     -p 3000:3000 \
     -e GEMINI_API_KEY=your_api_key \
     -e DATABASE_URL=file:./dev.db \
     --name kg-builder \
     knowledge-graph-builder
   ```

3. **Access**
   - Open `http://localhost:3000`

### Docker Compose (Alternative):

Create `docker-compose.yml`:

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - GEMINI_API_KEY=${GEMINI_API_KEY}
      - DATABASE_URL=file:./dev.db
    volumes:
      - ./prisma:/app/prisma
```

Run:
```bash
docker-compose up -d
```

## Option 4: Netlify

### Steps:

1. **Install Netlify CLI**
   ```bash
   npm install -g netlify-cli
   ```

2. **Build**
   ```bash
   npm run build
   ```

3. **Deploy**
   ```bash
   netlify deploy --prod
   ```

4. **Add Environment Variables**
   - In Netlify dashboard: Site settings → Environment variables
   - Add `GEMINI_API_KEY`

## Upgrading to PostgreSQL (Production)

### 1. Update Prisma Schema

Change in `prisma/schema.prisma`:
```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

### 2. Update Environment Variable

```env
DATABASE_URL="postgresql://user:password@host:5432/dbname"
```

### 3. Run Migrations

```bash
npx prisma migrate dev
npx prisma generate
```

### 4. Redeploy

## Environment Variables Reference

| Variable | Required | Description |
|----------|----------|-------------|
| `GEMINI_API_KEY` | Yes | Google Gemini API key from [makersuite.google.com](https://makersuite.google.com/app/apikey) |
| `DATABASE_URL` | Yes | Database connection string (auto-set for SQLite) |
| `NODE_ENV` | No | Set to `production` for production builds |

## Troubleshooting

### Issue: "Gemini API not connected"
- Check if `GEMINI_API_KEY` is set correctly
- Verify API key is valid at Google AI Studio
- Check API quota limits

### Issue: "Database connection failed"
- For SQLite: Ensure write permissions
- For PostgreSQL: Verify connection string
- Run `npx prisma migrate deploy`

### Issue: "Build fails"
- Clear `.next` folder: `rm -rf .next`
- Reinstall dependencies: `npm ci`
- Check Node.js version (18+)
